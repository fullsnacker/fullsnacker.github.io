As all web developers do, I have a set of tools I use on every web development project.  So I thought I would share mine with you. These are tools that I can say with all honesty have greatly increased my efficiency. I should mention that I work primarily with Apple products so some of these tools may not be available for PC users. However, there are equilvalents out there.

Visual Studio Code

Obviously a code editor is an essential for any programming development stack but I tried a lot of the popular code editors and VS Code is my favourite by far. I started out on Sublime Text and used it all through college and the first two years of my web development career. Then I switch companies and they used WebStorm. I liked it a lot but it wasn’t free so when I left that company I started trying other editors. I used atom for a while but found it to be a little slow and clunky. Then there was a large movement to VS Code so I thought I’d see what all the fuss was about. And after using it for a week I knew why. The speed, the plugins and the built in terminal are just a couple of my favourite features. It’s now an essential tool in my development belt. 

Sass (Syntactically Awesome Style Sheets)

Now as a web developer I spent a lot of time writing CSS. When I was at an FITC (Future Innovations Technology Conference) in Toronto quite a few years ago I went to a talk on Sass and it changed my development stack forever. I’m not going to go into huge detail about all its features but the site says

Sass is the most mature, stable, and powerful professional grade CSS extension language in the world.

Sass is a CSS preprocessor, an extension of CSS that adds incredible functionality. Sass is compiled through a tool called Compass into CSS. The amount of time writing in Sass has saved me is incalculable.

There are other CSS preprocessors like LESS or Stylus but Sass is mine and most people’s choice.

Bourbon Neat

After integrating Sass into my development stack the question of what grid system should I use arose. I had been playing around with a few other grid systems but when I started tinkering with Neat I immediately knew this was the one. Bourbon Neat is 

A lightweight and flexible Sass grid

Its simple and doesn’t use any flex-box (flex-box wasn’t always stable) but still has incredible customization so it fits every project I build. It uses Sass mixins to set column widths and offsets so I never need to add loads of classes to my HTML. 

Thoughtbot is the development team that created Neat and they also have some other great tools to checkout like Bourbon and Bitters (there’s a theme, yes).

Codekit

This is the ultimate tool! Codekit‘s main purpose is to compile and compress languages down for faster, better development but it does so much more than that. Its browser refresh feature is one of my favorites. I make a change in my code and it automatically recompiles and refreshes my browser. I know that’s not revolutionary but it makes it so simple to get back into a project. Just drag your project in and within a couple clicks, you are set up with Sass, Compass, Bourbon Neat etc. 

Another great feature is that it can take multiple Sass or Javascript files and compile them into one compressed file. A great way to increase load times for your website. 

Codekit has piles of other features that I’ll let you discover for yourself. I will mention that this is a Mac-only product. If you know of something similar for PC I would love it if you mentioned it in the comments.

MAMP

MAMP is the final piece of the puzzle. According to the website

MAMP is a free, local server environment that can be installed under macOS and Windows with just a few clicks. 

MAMP turns your computer into a local server so that you can run and test PHP. 

Most of my clients want to run the content for their websites so natural I use WordPress as the CMS (content management system). This means that I need to be able to develop in PHP and run MySQL databases. And that’s exactly way MAMP allows me to do.

Now MAMP does have a paid version but I have never felt the need to pay for it when the free version does exactly what I need. From my understanding, the paid version is just a better interface but if you feel that the pro version is essential, let me know it the comments.

MAMP is a mac product but I know there are PC tools that do the same, such as LAMP.

Those are my top 5 essential web development tools. If you have other tools that you use on every project I would love to hear about them. Always down to add or replace tools with better ones. Let’s keep making our code DRYer and our development time shorter!

When I started to get into the coding world I immediately started looking for good places to learn online. I’ve never been a book kind of person and I think learning to code from books is a little “clunky” (Leave a comment at the bottom of your opinion). So I surfed around and found:

Codecademy

This is an amazing resource if you’re looking to get started in the programming world because it assumes you have no prior knowledge. It has an in-browser editor, step-by-step instructions and good explanations and hints for all the lessons. And best of all, its free! There is a pro version available that gives you access to more help. This site is how I got started in programming and its always my first recommendation.

Pluralsight

So after I had been working as a web developer for a year a software company asked if I would join their team as front-end lead. Not knowing anything about software developing I naturally took the job. And that’s when they recommended I use Pluralsight to get up to speed. It was a monthly membership fee of $15 and I could go through as many tutorials as I wanted and I did. The courses were thorough and clear about all the concepts, terms and syntax. I was able to get up to speed with the company in a matter of days! 

Udemy

Now I saved this for last because it’s my favorite and I still use it. I can honestly save I have never learned so much from a website than I have from Udemy. The rating system and recommended courses they gave were exactly what I wanted and although you pay per course, the courses are hugely extensive, thorough, and easy to follow. Stephen Grider is a personal favorite instructor of mine. He really walked me through every step, not leaving any stone unturned. A lot of the courses have expectations that you have some knowledge of the subject but there are courses that will take you from the ground up. Udemy is my top recommendation for learning to code. 

BONUS: Youtube

Learncode.academy

There is a world of resources out there for learning to program. All you have to do is Google. Youtube is another place to learn and there are piles of channels dedicated to creating tutorials. Learncode.academy was a channel that I found myself frequently visiting for great tutorials and a range of coding subjects and frameworks.

Computer

Obviously to have a computer setup you need to have a computer to start with. And I use a laptop so that’s really all I “need” but if I’m being honest to work comfortably I need more. So I started my setup with a MacBook Pro 15″. I was able to get one with a touch bar. People ask whether I like the touch bar or not and honestly, it’s not a big deal to me. I like it and some of the features are cool but not revolutionary to my workflow.

Monitors

I could just work off a laptop but I need more screen space than 15 inches. So I got 2 Samsung 27″ LED monitors. They aren’t 4k which I kind of wish they were but when I’m just writing code I don’t need the ultra high definition. In the future I’d love to upgrade.

Monitor Stand

So I had both monitors just sitting on my desk and the stands they were on took up half of it so I immediately head to Amazon for a solution. Right away I found Vivo and their incredible line of products. I purchased my Vivo dual monitor mount and could not be more pleased with a product. I got so much desk space back and actually helped with my posture because I was able to raise the screen to a better height. Truly a king-pin of my setup.

HDMI to USB-C

I get asked a lot how I am able to have dual monitors with a MacBook Pro and the answer is super simple, HDMI to USBC cable. I just plug the HDMI end into the monitor and the USBC end into my Mac and it works. I go into Settings > Display and adjust the layout and we’re rolling.

Keyboard

Another cornerstone of my setup is my keyboard because as we all know, a programmers number one tool is their keyboard. I had been using a Razor Black Widow Tournament Edition for quite some time and really enjoyed it. Then I moved to an X-bow Ergonomic Keyboard for a couple months until I switched to what I use now, the Vortex Race 3. I got it with silent red switches because I love reds but I also work in a co-working office (so a loud keyboard isn’t very welcome). I can honestly say this is my keyboard for life. I know many other programmers as well that swear by it.

Key Caps

But obviously, I can just leave something with the stock setup. Even though the original keycaps are amazing I had to customize. So being the very extra person I am, I got golden keycaps to really “pimp out” my keyboard. Being extra is a necessity for me.

Mouse

Most programmers that pride in the fact that they don’t need to use a mouse at all but as I also do a lot of graphic design, its used an equal amount. I’ve had my magic mouse for over 9 years and it works flawlessly which I think is a huge testament to the quality of the product. Having the touch features and being able to swipe through multiple desktops has been so essential to my workflow. 

Wrist Pads

Keeping my setup ergonomic is key because I spend full days at my desk. I have struggled in the past with shoulder numbness and carpal tunnel so having proper wrist support is a must. I use Grifiti Fat Wrist Pads for my mouse and keyboard and I haven’t had any pain.

Headphones

As all good programmers do, I need quiet, undistracted focused to be able to code to the best of my abilities. But as I mentioned before, I work in a co-working space, so silence is rare.  The solution: headphones. And there is no better solution that AirPods. I was a skeptic of them at first but my friends kept raving about them so I took a chance and got some. I was not disappointed. They rarely leave my ears. I can wear them for a full day and my ears don’t get sore at all (which is usually a problem for me). 

Standing Mat

The final piece to my setup is my standing mat. I have a sit-to-stand desk which has been incredible for productivity and mood but having high-arched feet, they get fatigued quickly. The ErgoDriven Standing Mat has an amazing design to help counter standing fatigue and improve posture. I am able to move around on it and even massage my feet while I work. An essential piece to my computer setup.

If you guys have more questions about my setup or thoughts on any of these products I’d love to hear about them in the comments. My setup is constantly changing so I’ll keep you updated when I switch things up.
